Using MIPCL from other programming languages

We recommend using MIPCL-PY for rapid development and writing prototypes, as well as for teaching mixed integer programming.

The only reasonable way of using MIPCL in other languages is

  • to implement optimization problems in C++ via classes derived from CMIP or CLP,
  • and then write interfaces for using those classes from the target language.

Java Interface to MIPCL

If you are a Java programmer, and, for any reason, you decided not to follow the above schema, you can use the MIPCL-JAVA — a module that implements the Java Native Interface (JNI) to the MIPCL. MIPCL-JAVA implements only a part of the MIPCL API. Complex applications requiring redefining MIPCL virtual functions — say, those needed for generating problem specific cuts — cannot be implemented using MIPCL-JAVA. Efficiency issues are the main reason for such restrictions. In addition, debugging complex programs that mix languages is a real headache.

MIPCL-JAVA comprises of two packages: by.bsu.JVmipcl and by.bsu.JVmipshell. The former contains two classes — LP and MIP — that are wrappers, respectively, for the CLP and CMIP, which are MIPCL main classes. You can use LP and MIP for modeling simply structured LPs and MIPs. But for more complex (from modeling point of view) optimization problems, it is preferable to use two classes — LPshell and MIPshell — from by.bsu.JVmipshell.
